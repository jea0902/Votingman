# Goal of Votingman

## 우리가 만들고 싶은 것

Votingman의 목표는 단순한 "투표 웹사이트"가 아니라, 예측과 정보가 실제 수익으로 이어지는 의사결정 플랫폼이다.

핵심 목표는 아래 3가지다.

1. Polymarket처럼 예측 거래소를 만든다.
2. 진짜로 돈을 벌 수 있는 정보 공유 플랫폼을 만든다.
3. AI 에이전트들로 돈을 벌 수 있는 정보 공유 플랫폼을 만든다.

---

## 문제 인식 (현재 상태)

- 현재 형태의 투표 서비스는 사용자가 "왜 지금 참여해야 하는지"가 약하다.
- 참여 방식이 다소 무겁고, 즉시 이해 가능한 가치(재미, 보상, 인사이트)가 부족하다.
- 결과적으로 재방문/습관화가 어려운 구조다.

즉, "정교한 기능"보다 먼저 "즉시 참여 가능한 초간단 경험"이 필요하다.

---

## 제품 원칙

### 1) 3초 이해, 1탭 참여
- 첫 화면에서 질문과 선택지만 보이고, 길게 읽지 않아도 참여 가능해야 한다.
- 복잡한 입력은 숨기고 기본 행동(예측/참여/공유)만 전면 배치한다.

### 2) 의견이 아니라 "수익 가능한 신호"
- 투표는 감상평이 아니라, 실제 포지션/시장과 연결되는 신호여야 한다.
- 각 참여는 "왜 이 선택이 돈이 될 수 있는지" 근거와 함께 기록되어야 한다.

### 3) 사람 + AI 협업
- 사람은 현장감 있는 아이디어와 맥락을 제시하고,
- AI는 데이터 근거, 리스크, 실행 시점, 기대값을 구조화한다.

### 4) 신뢰 가능성 우선
- 성과는 과장 없이 검증 가능한 지표(적중률, 기대수익, 최대손실, 리스크 노트)로 제시한다.

---

## 핵심 아이디어 제안

## A. "유튜브처럼" 초간단 참여 흐름
- 카드형 피드: 한 화면에 질문 1개 + 선택지 2개(UP/DOWN, YES/NO).
- 스와이프/탭 1번으로 참여 완료.
- 각 카드에 "마감 시간", "현재 다수 의견", "AI 한줄 요약"만 표시.

예시:
- "오늘 BTC 종가가 1% 이상 오를까?"
- [Yes] [No]

복잡한 글쓰기 없이도 참여가 쌓이게 만든다.

## B. 예측을 "시장"으로 연결
- 예측 질문마다 가상 포인트 또는 소액 실전 모드(규제/법무 검토 후)를 연결.
- 마감 시 정산 로직을 자동화하고, 결과/손익을 공개 대시보드로 제공.
- "정답률"이 아니라 "기대값 기반 성과"를 중심 지표로 삼는다.

## C. AI 에이전트 기반 인사이트 레이어
- 각 질문마다 AI가 자동 생성:
  - 찬성/반대 핵심 근거 3줄
  - 주요 리스크 2개
  - 무효화 조건(언제 관점 철회할지)
- 사용자는 AI 리포트를 보고 1탭으로 참여.
- 고급 사용자는 AI 설정(보수형/공격형/단타형) 선택 가능.

## D. "돈 되는 정보 피드"로 전환
- 단순 투표 피드가 아니라 "트레이드 아이디어 피드"로 재구성:
  - 아이디어 제목
  - 진입 조건
  - 청산 조건
  - 손절 기준
  - 근거 데이터
- 투표는 "이 아이디어를 따라갈지 여부"의 빠른 신뢰 신호로 사용.

## E. 참여 장벽 제거 장치
- 로그인 전에도 읽기/미리보기 가능, 참여는 소셜 간편 로그인 1회로 끝.
- 첫 참여 보상(포인트/뱃지/리더보드) 제공.
- 알림: 내가 참여한 예측의 마감 10분 전, 결과 확정 시 자동 알림.

---

## 제안 기능 로드맵 (MVP -> 확장)

### Phase 1: 초간단 참여 MVP
- 카드형 예측 피드
- 2지선다 1탭 참여
- 마감/결과/정산 자동 표시
- 기본 리더보드(적중률, 참여수)

### Phase 2: 수익성 강화
- 아이디어별 기대값/리스크 표시
- 손익 추적(가상 포인트 기준)
- AI 한줄 근거 + 리스크 경고

### Phase 3: AI 에이전트 플랫폼화 (Research -> Auto Trade -> SNS)

- AI 에이전트별 성과 공개(승률, 손익, MDD, 샤프, 최근 30일/90일)
- 에이전트 팔로우/복제(시그널 구독 + 세팅 템플릿 복제)
- 사용자 + AI 공동 포트폴리오(사람 판단 + AI 실행 분리 기록)

#### 3-1) 핵심 제품 구조: "트레이딩 SNS"
- 리서치: AI가 뉴스/온체인/펀딩비/기술지표를 요약하고 매매 가설 생성
- 자동매매: 사용자 승인 기반 반자동 또는 완전자동 모드 제공
- 결과 공유: 백테스트/실전 성과를 카드 형태로 업로드하고 토론

즉, "아이디어 게시판"이 아니라 "검증 가능한 전략 기록 SNS"로 설계한다.

#### 3-2) 유저 역할
- 크리에이터(전략 제공자): 에이전트 세팅, 전략 논리, 백테스트/실전 성과를 공개
- 팔로워(구독자): 크리에이터를 팔로우하고 시그널/세팅/인사이트를 구독
- 일반 사용자: 무료 피드 열람 + 일부 전략 체험 + 랭킹 확인

#### 3-3) 에이전트 페이지 필수 정보 (신뢰 확보)
- 전략 설명: 어떤 시장/시간프레임/리스크를 다루는지
- 백테스트: 기간, 거래 수, 승률, MDD, 수수료/슬리피지 반영 여부
- 실전 성과: 월별 손익률, 누적 손익, 최근 드로우다운
- 리스크 규칙: 손절/익절/진입금지 조건, 비정상장 대응 규칙
- 검증 상태: 모의투자만/소액 실전/장기 실전 트랙레코드 배지

#### 3-4) 코인 1차 MVP 수익모델 (레퍼럴 수익 쉐어 중심)

구독보다 먼저, 해외 거래소 레퍼럴 수익을 전략 제공자와 플랫폼이 나누는 구조를 기본 모델로 채택한다.

- 대상: 코인 선물/현물 거래소 제휴(예: Binance, Bybit, OKX, Bitget 등)
- 발생 수익: 유저 거래로 발생한 거래소 리베이트/레퍼럴 수익
- 분배 대상: 전략 제공자(크리에이터) + Votingman 플랫폼

초안 분배안(제안):
- 기본안: 크리에이터 60% / 플랫폼 40%
- 성장안: 월 실적 구간별 크리에이터 우대(예: 상위 티어 70:30)
- 신규 유입 보너스: 첫 30일은 크리에이터 추가 +5% 인센티브

정산 원칙:
- 정산 주기: 월 1회 (익월 초 정산)
- 정산 기준: 거래소에서 확정된 순 리베이트(취소/환수 반영 후)
- 최소 정산 금액: 운영비 고려한 하한선 설정(예: 50 USDT)
- 환율/수수료: 지급 통화와 송금 수수료 정책을 사전 고지

추적(Attribution) 방식:
- 크리에이터별 전용 레퍼럴 링크/코드 발급
- 링크 클릭 -> 가입 -> 거래 발생까지 이벤트 체인 저장
- 복수 기여가 있는 경우 Last Touch 기본 + 어시스트 로그 보관

신뢰/운영 장치:
- 크리에이터 대시보드: 클릭/가입/거래/예상정산/확정정산 분리 표시
- 사용자 고지: "제휴 수익이 발생할 수 있음" 명시(투명성)
- 부정행위 방지: 셀프거래/비정상 거래 패턴/어뷰징 계정 차단
- 컴플라이언스: 국가별 제휴·광고·투자자문 규정에 맞춘 문구/차단 정책

향후 확장:
- 2차: 카피트레이딩 실행 수수료(성과 연동) 추가
- 3차: 전략 마켓플레이스(유료 템플릿/봇 설정 판매) 추가

#### 3-5) "자랑" 기능을 건강하게 만드는 장치
- 월 수익률 인증 카드: 백테스트/실전 여부를 명확히 구분
- 성과 과대포장 방지: 수익률 옆에 MDD, 거래 횟수, 표본 기간을 동시 표시
- 리더보드는 단일 수익률이 아니라 "위험조정 수익" 중심 정렬
- 손실 공개 인센티브: 손실 복기 포스트도 평가/가점 반영

#### 3-6) 팔로우/복제 UX
- "팔로우": 작성자 업데이트, 신규 전략, 성과 변동 알림
- "전략 복제": 설정값 1클릭 가져오기 (자본/리스크만 개인화)
- "시뮬레이션 먼저": 실전 적용 전 7일 모의 성과 자동 제공
- "가드레일": 과도 레버리지, 거래소 장애, 비정상 슬리피지 시 자동 중지

#### 3-7) 구현 우선순위 (실행 가능한 단위)
1) 에이전트 프로필 + 성과 카드 피드
2) 백테스트 결과 업로드 표준 포맷(JSON + 스냅샷)
3) 실전 연동(읽기 전용 API) 기반 월별 성과 인증
4) 거래소 제휴 연동 + 레퍼럴 추적(링크/코드/어트리뷰션)
5) 정산 대시보드(예상/확정 정산, 크리에이터 배분 로직)
6) 전략 복제 + 모의투자 샌드박스
7) 자동매매 연동(사용자 승인 기반 단계적 오픈)

---

## 성공 지표 (North Star 후보)

- 참여 전환율: 피드 조회 -> 1탭 참여 전환
- 재방문율: D1 / D7 / D30
- 유효 참여율: 마감까지 유지된 참여 비율
- 정보 품질 지표: AI 근거 조회율, 근거 기반 참여 비율
- 수익 지표: 사용자 평균 기대수익, 실현 손익(가상/실전 분리)

---

## 결론

Votingman은 "투표 서비스"를 넘어,  
"초간단 참여 UX + 검증 가능한 수익 정보 + AI 실행 보조"를 결합한  
예측 기반 수익 정보 플랫폼으로 진화해야 한다.

핵심은 복잡함을 줄이고, 참여를 습관으로 만들며,  
결국 사용자에게 "이 앱이 실제로 돈 되는 판단에 도움된다"는 확신을 주는 것이다.
