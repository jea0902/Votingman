# 홈 화면 레이아웃 명세

보팅맨 홈 메인 영역의 구조·탭·디자인 방향을 정리하는 문서입니다. 확정된 내용은 여기에 기록하고, 후보안은 검토 후 반영합니다.

---

## 1. 목적·범위

- **대상**: 홈 메인 영역 (전체 3단 구성 ~ 비전 문구 ~ 탭 ~ 탭별 콘텐츠).
- **목적**: 레이아웃·탭 구조와 디자인 방향을 명세로 두고, 구현·기획 시 일관되게 참고.

---

## 2. 현재 홈 화면 전체 구조 (2026-02 기준)

### 2.1 반응형 분리

- **작은 화면 (lg 미만)**: 단일 컬럼(`flex flex-col`), 7개 블록을 **DOM 순서대로** 위에서 아래로 노출.
- **큰 화면 (lg 이상)**: 3열 그리드(`grid-cols-[1fr_2fr_1fr]`), 왼쪽 25% | 가운데 50% | 오른쪽 25%.

### 2.2 작은 화면 블록 순서 (위 → 아래)

| 순서 | 블록 | 설명 |
|------|------|------|
| 1 | 메인 문구 | 비전 제목 + 부제 (세로 min-h 20vh) |
| 2 | 비트멕스 리더보드 TOP5 | TopRankersBoard |
| 3 | 코인 유튜버 실시간 포지션 | InfluencerPositions |
| 4 | 시장 탭 카드 | MarketTabCards (비트코인 \| 미국 주식 \| 한국 주식) |
| 5 | 인간 지표(투표) + 플레이스홀더 | HumanIndicatorSection + SECTION_LABELS |
| 6 | UserInfoCard | 시즌 랭크 / 로그인 유도 |
| 7 | 보팅맨 TOP20 랭커 실시간 포지션 | MarketTop20Positions |

- 블록 간 간격: `gap-8`.

### 2.3 큰 화면 3열 구조

| 열 | 비중 | 내용 |
|------|------|------|
| **왼쪽** | 1fr (25%) | UserInfoCard + MarketTop20Positions. 상단 `pt-[20vh]`, 두 카드 사이 `gap-3`. sticky. |
| **가운데** | 2fr (50%) | **한 개의 `<main>`** 안에서 `flex flex-col gap-4`: (1) 메인 문구 `min-h-[20vh] justify-end`, (2) MarketTabCards, (3) HumanIndicatorSection + 플레이스홀더. |
| **오른쪽** | 1fr (25%) | **한 개의 `<aside>`** 안에서 `flex flex-col gap-4`: (1) InfluencerPositions, (2) TopRankersBoard. |

- 가운데·오른쪽은 **한 컬럼씩 하나의 flex 컨테이너**로 묶어, 그리드 행 높이에 따른 빈 여백·카드 누락을 방지.

### 2.4 전체 비중 (가로 배치, lg 이상) — 요약

| 구역 | 비중 | 설명 |
|------|------|------|
| **왼쪽** | 25% | UserInfoCard, 보팅맨 TOP20. 상단 20vh 패딩, 카드 간 gap-3. |
| **가운데(메인)** | 50% | 메인 문구(20vh) → 시장 탭 → 투표. flex로 붙여서 세로 간격 최소화. |
| **오른쪽** | 25% | 코인 유튜버 포지션 → 비트멕스 리더보드. |

### 2.5 레이아웃 이슈·해결 요약 (2026-02)

- **문제 1 (세로 여백)**: 그리드에서 1행 높이가 왼쪽 열(두 카드)에 맞춰 커지면서, 가운데 문구(20vh) 아래에 빈 공간이 크게 생김. 이후 2행 높이도 오른쪽/왼쪽 열에 맞춰져 시장 탭 ↔ 투표 사이에도 큰 여백 발생.
- **해결 1**: 1행을 `grid-rows-[20vh_auto_auto]`로 고정·문구를 `justify-end`로 아래 정렬해도, 2행이 다른 열 때문에 늘어나는 구조적 한계가 있음.
- **문제 2 (비트멕스 카드 누락)**: 오른쪽 열을 1행(유튜버)·2행(비트멕스)으로 나누어 두어, 뷰포트/행 높이에 따라 비트멕스 카드가 안 보이는 경우 발생.
- **최종 해결**: 큰 화면만 **모바일용·데스크톱용 레이아웃을 DOM으로 분리**. 작은 화면은 기존 7블록 flex-col 유지(`lg:hidden`). 큰 화면은 `hidden lg:grid`로 3열만 사용하고, **가운데·오른쪽을 각각 한 개의 flex 컨테이너로 묶음**. 가운데는 문구 → 시장 탭 → 투표를 `gap-4`로만 이어 붙이고, 오른쪽은 유튜버 → 비트멕스를 한 컬럼에 세로로 배치해 둘 다 항상 노출되도록 함.

### 2.6 왼쪽 영역 (UserInfoCard) — 유저 정보 카드 (확정)

- **컴포넌트**: `UserInfoCard` (lol.ps 스타일 참고).
- **비로그인**: 카드 안에 로그인·회원가입 버튼. “시즌 랭크” 헤더 + 안내 문구.
- **로그인**: 시즌 랭크, 티어·MMR, 승률, 전적(승/패), 일별 MMR 추이 그래프 영역(추후 연동).
- **배치**: 왼쪽 25% 영역 안에서 `justify-center`로 정중앙, 카드 자체는 `max-w-[280px]`. lg 이상에서 sticky.

### 2.7 가운데(메인) 구성 (위 → 아래)

| 순서 | 영역 | 설명 |
|------|------|------|
| 1 | **비전 문구** | 제목 + 부제. 파란 제목(#3b82f6), 노란 부제(#fbbf24). 버핏 원픽과 동일 스타일. |
| 2 | **3개 탭** | 비트코인 \| 미국 주식 \| 한국 주식. 4.1안(큰 블록 카드) 적용. 한 번에 하나만 선택. |
| 3 | **탭별 콘텐츠** | 선택한 탭에 해당하는 인간 지표(시장별 투표 카드) + 공통 플레이스홀더 섹션들. |

### 2.8 비전 문구 (확정)

- **제목**: "탈중앙화 시장 예측 배팅 플랫폼" (따옴표 없음)
- **부제**: "투자자들이 코인을 배팅해 투자 심리를 반영한 인간 지표" (실제 문구는 운영에서 수정 가능)
- **스타일**: 제목 `text-[#3b82f6]`, 부제 `text-[#fbbf24]`. 반응형 폰트 크기 적용.

### 2.9 탭 구조 (확정)

- **탭 3개**: 비트코인(btc) | 미국 주식(us) | 한국 주식(kr)
- **탭 디자인**: **4.1안(풀스크린 탭 스위처)** 적용. 큰 블록 카드 형태, 선택 시 해당 카드 전면·확대, 나머지 뒤로/축소.
- **동작**: 탭 클릭 시 해당 탭만 활성, 아래 콘텐츠가 해당 시장만 표시.
  - 비트코인: 인간 지표 비트코인 카드 1개
  - 미국 주식: 나스닥·S&P500 카드 2개
  - 한국 주식: 코스피·코스닥 카드 2개
- **접근성**: `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `aria-controls` 등 사용.

### 2.10 배경 (확정)

- 상단 중앙에 파란 계열 radial gradient (ellipse). `rgba(59,130,246,0.08)` 수준.

## 3. 탭 디자인 후보 (게임스럽게 — 사용자 요청 3안)

검토 후 선택·조합하여 확정 예정. (현재는 4.1안 적용.)

### 3.1 아케이드 / 레트로 픽셀

- **컨셉**: 오래된 게임기·아케이드 캐비닛, 픽셀 폰트·테두리.
- **탭**: 두꺼운 테두리(검정/네온), 선택 탭만 밝은 색(네온 그린·옐로우·시안) 강조. 픽셀/비트맵 폰트.
- **참고**: itch.io 인디 게임 메뉴, 레트로 RPG 스타일.

### 3.2 카드 / 덱 선택 UI

- **컨셉**: TCG·덱 빌더처럼 "카드를 고르는" 느낌.
- **탭**: 각 탭을 카드 한 장처럼(둥근 모서리, 그림자, 호버 시 살짝 상승). 선택 탭은 테두리 글로우(골드·퍼플) + 살짝 확대.
- **참고**: Hearthstone, Legends of Runeterra, Genshin Impact 메뉴 탭.

### 3.3 미니멀 게임 HUD / 스코어보드

- **컨셉**: e스포츠·레이싱 HUD처럼 깔끔한 "정보판".
- **탭**: 반투명 다크 배경 + 얇은 밝은 테두리. 선택 탭은 한쪽 세로 라인(액센트 컬러) 또는 하단 언더라인. 기하학적 산세리프, 필요 시 아이콘(비트코인·국기 등).
- **참고**: Valorant, Overwatch, LoL 클라이언트, Trackmania 탭.

## 4. 적용 중인 추천안 — 4.1 풀스크린 탭 스위처

- **컨셉**: 탭을 "큰 블록/카드"로 두고, 선택한 카드가 전면에 오는 방식. 모바일에서 스와이프·탭 전환에 유리.
- **탭**: 3개를 세로 또는 그리드로 큰 카드로 배치. 선택 시 해당 카드가 확대·앞으로, 나머지는 뒤로/축소. 스크롤 없이 한 화면에서 "시장 선택"이 메인 액션으로 보이게.
- **장점**: 진입 장벽 낮음, 터치 친화. "게임에서 월드/캐릭터 고르기" 느낌.
- **구현**: `MarketTabCards` 컴포넌트. 선택 카드 `scale-[1.02]`, `z-10`, 파란 테두리; 비선택 `scale-[0.98]`, `opacity-90`.
