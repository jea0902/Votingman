# 보팅맨(VotingMan) 비즈니스 기획서

> **랜딩 페이지 문구**
> "입으로만 떠드는 투자 전문가, 아직도 믿으십니까?"

투자 예측 시장 '보팅맨'은 삭제를 허용하지 않습니다.
수정 불가, 조작 불가. 여기서 증명된 자만이 '진짜 트레이더'입니다.

> **한 줄 요약**  
> “여기서 증명하면 진짜다” — **조작 불가능한 예측 박제**로 투자 시장에 **책임과 증명**을 부여하는, 대한민국 리얼 트레이더의 성지.

---

## 1. 핵심 가치 제안

| **한 문장** | 실제 돈(포인트)을 걸고 증명한 고수들의 **실시간 집단 신호**를 제공하고, 고수에게 경제적·명예적 보상을 주는 **검증 가능한 투자 정보 플랫폼** |
| **핵심 문구** | 복잡한 수식어 없이 **‘여기서 증명하면 진짜다’**라는 **사회적 합의**를 만드는 것이 이 플랫폼의 핵심 기능 |
| **비유** | 한번 쌓인 **실적은 수정·삭제할 수 없이 남는** 구조라, 이 플랫폼이 곧 조작 불가능한 **‘투자 실력 이력서’**가 됨 |

---

## 2. 문제 정의 (Problem)

### 2.1 시장의 구조적 문제

- **“입만 산 투자 전문가들이 너무 많다”**  
  유튜브, 텔레그램, 커뮤니티에서 “이거 오른다”, “저거 사라”라고 말하는 사람은 넘치지만, **말이 맞았는지 틀렸는지 검증할 방법이 없다.**

- **책임과 증명이 없는 시장**  
  시간이 지나면 **틀린 글은 삭제되고, 맞은 글만 남아 성지 글이 된다.**  
  → **과거 예측이 맞았는지 나중에 투명하게 확인할 수 없다**는 것이 이 시장의 가장 큰 한계다.

### 2.2 사용자 니즈

- **“다 필요 없고, 믿을 수 있는 돈이 되는 정보”**  
  전문가도 뉴스도 믿기 어려운 시대에, **실제 돈(또는 포인트)을 걸고 증명해낸 고수들의 실시간 집단 신호**가 필요하다.
- 고수들에게 **경제적·명예적 보상**을 주어, 플랫폼이 “검증된 정보의 허브”로 자리 잡게 한다.

---

## 3. 타겟 고객

| 세그먼트      | 니즈                                            | 플랫폼 가치                                                    |
| ------------- | ----------------------------------------------- | -------------------------------------------------------------- |
| **중수 이하** | 신뢰할 수 있는 투자 정보를 찾는 일반 투자자     | 숫자로 검증된 고수의 포지션·승률을 보고 참고                   |
| **중수 이상** | 다른 고수들의 포지션을 보고 싶은 투자자         | 돈이 걸린 포지션 = 가장 확실한 예측 방향으로 활용              |
| **고수**      | 자신의 판단을 의심할 때 다른 투자자 포지션 참고 | 실적이 쌓여 **검증 배지·메인 노출**로 자신을 알릴 수 있는 공간 |

---

## 4. 솔루션

### 4.1 핵심 메커니즘: **삭제 불가능한 예측 영수증**

유저가 **상승/하락에 투표하는 순간**, 시스템은 다음 정보가 담긴 **하나의 ‘카드’**를 생성하고 **유저 프로필에 영구 박제**한다.

| 항목               | 설명                                        |
| ------------------ | ------------------------------------------- |
| **예측 대상**      | 비트코인 일일/4시간/1시간/15분 가격 방향 등 |
| **예측 시점 가격** | 투표 시점의 시장 가격                       |
| **예측 방향**      | 상승 / 하락                                 |
| **결과 확정일**    | 예: 12시간 후, 해당 봉 종료 시점 등         |

**중요:**

- **투표 생성 후 수정·삭제 불가** → 조작 불가 구조
- **투자 자문이 아님** → 매수/매도를 직접 추천하지 않고, **참여자들의 포지션·승률 같은 결과만 보여주는** 플랫폼

### 4.2 자동 성과 기록

- **결과 확정일**이 되면, **실제 시세**를 가져와 **성공/실패를 자동으로 판정**
- 유저 프로필에 **총 예측 횟수, 전적, 예측 시장별 승률** 등이 **숫자로 고정**

### 4.3 돈이 걸린 구조 (파리 뮤추얼)

- 투표로 **승리한 사람**이 **패배한 사람이 건 만큼의 비율**을 가져가는 구조
- “말”이 아니라 **돈(포인트)이 걸린 포지션**이라서, 가장 확실한 예측 방향이 됨

## 5. 차별화 — 왜 보팅맨인가?

| 차별점                        | 설명                                                                                                                                                                        |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **가짜 전문가의 지속 불가능** | 잘한다고 **주장만 하는 사람**이 아니라, **숫자로 증명된 사람**만 살아남는 구조. **한번 남은 실적 기록** = 조작 불가능한 **투자 실력 이력서**                                |
| **한눈에 보는 포지션**        | 유튜버 TOP3, 바이낸스/비트멕스 TOP5, **보팅맨 랭커 TOP5** 등 **고수와 다수 참여자들의 실시간 포지션**을 한 줄로 제공                                                        |
| **참여자들의 방향을 한곳에**  | 승률·수익률이 높은 고수들이 **어디를 보고 있는지**를 모아서 **쓸모 있는 정보**로 제공                                                                                       |
| **타점 시각화(추가)**         | 예측한 순간의 **차트 화면**을 저장해 둠 → 고수에게는 **실적 모음**, 초보에게는 **공부 자료**                                                                                |
| **실력 인정 프로그램(추가)**  | 적중률이 검증된 유저에게 **메인 화면 노출**, **‘검증됨’ 배지**, **유튜브·SNS에 걸 수 있는 공식 프로필 링크** 제공 → 플랫폼을 **자기 실력을 증명하는 공식 창구**로 쓰게 유도 |

## 6. 왜 믿을 수 있나? — 기록의 신뢰성

### 6.1 최우선 원칙

**“예측 기록을 조작할 수 없게 만들고, 그걸로 유저가 믿을 수 있게 하는 것”**이 이 플랫폼의 **가장 중요한 설계 원칙**이다.

### 6.2 처리 순서 (한번 투표하면 이렇게 흐른다)

```
투표 생성 → 투표 시각 기록 → 실제 시세와 비교 → 자동 정산 → 유저별 승·패 판정 → 순위 반영
```

### 6.3 리스크 & 대응

| 리스크                                | 대응                                                                                                                                 |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **거짓 기록·여러 계정**으로 승률 조작 | 카카오 로그인만 사용, 계정당 VTC 1회 지급 등으로 여러 계정 만들기 억제                                                               |
| **단 한 번의 조작 사고**로 신뢰 붕괴  | **기록 조작이 불가능하다는 설계**와 **그걸 알리는 메시지**가 핵심 — 조작이 안 된다는 점을 명확히 하고, 누구에게나 설명 가능하게 유지 |

**참고:**  
직접적인 외부 사례는 없으며, **‘예측에 참여해 결과로 검증한다’는 방식**과 **기록의 투명성**을 바탕으로 한 아이디어이다.
